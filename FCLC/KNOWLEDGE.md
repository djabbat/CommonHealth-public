## Federated Learning (FL)

- **FedAvg**: усреднение весов моделей от клиентов. Проблема: heterogeneous data.
- **FedProx**: добавляет проксимальный член для гетерогенных данных.
- **Convergence**: при non-IID данных требует >100 раундов для стабильного обучения.

## Differential Privacy (DP)

- **Gaussian mechanism**: добавляет N(0, σ²) к градиентам. σ = C·√(2ln(1.25/δ))/ε
- **Моменты**: R'enyi DP (RDP) даёт более точный композиционный анализ.
- **Практика**: ε=1.0, δ=1e-5 — стандарт для медицинских данных (Google, Apple).

## SecAgg+

- Протокол безопасной агрегации без сервера, видящего индивидуальные обновления.
- Поддерживает dropout ≤30% клиентов.
- Вычислительная сложность: O(n log n) на клиента.

## Медицинские данные в FL

- GDPR Art. 9: медицинские данные — специальная категория.
- DPIA обязателен при использовании медицинских данных в ЕС.
- Синтетические данные (CTGAN, TVAE) — допустимый обходной путь.

## LEVF (Local Empirical Value Function)

- Концепция: вклад участника пропорционален улучшению модели на тестовом наборе.
- Метод Шэпли (Shapley values) — теоретически точный, но экспоненциально сложный.
- Практика: gradient-based attribution (GTG-Shapley).
