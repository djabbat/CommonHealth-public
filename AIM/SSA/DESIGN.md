# DESIGN.md — Системный Синдромальный Анализ (SSA) полного клинического анализа крови

## 1. Цель и обоснование

Проект **SSA** реализует формализованный переход от дискретного набора CBC-параметров (28 показателей) к клинически интерпретируемому вектору синдромальных паттернов. Методология основана на классических гематологических классификациях (Wintrobe 1961, Bessman 1983, Mentzer 1973) и современных консенсусных референсных интервалах (ICSH 2014, WHO 2011, BCSH 2017). Движок оперирует пятизонной шкалой (L2, L1, L0, H1, H2), что позволяет детектировать не только клинически значимые отклонения, но и пограничные состояния, часто упускаемые бинарной логикой.

## 3. Слой данных (Data Layer)

### 3.1 `ranges.json` – пятизонные границы

Каждый параметр имеет набор записей, дифференцированных по полу (`sex`), возрастной группе (`age`, `age_min`, `age_max`), статусу беременности (`pregnancy`). Для каждой записи определены пять числовых интервалов:

- `L2`: ниже критического минимума (менее 2.5-го перцентиля популяции с учётом клинического порога)
- `L1`: между критическим минимумом и нижней границей нормы
- `L0`: оптимальный диапазон (между 2.5-м и 97.5-м перцентилями, либо узкий клинический оптимум)
- `H1`: между верхней границей нормы и критическим максимумом
- `H2`: выше критического максимума

Пример для гемоглобина (мужчины 18+):

```json
{
  "id": "HGB",
  "unit": "g/L",
  "ranges": [
    {
      "sex": "male",
      "age_min": 18,
      "age_max": null,
      "pregnancy": false,
      "zones": {
        "L2": {"max": 80},
        "L1": {"min": 80, "max": 134},
        "L0": {"min": 135, "max": 175},
        "H1": {"min": 175, "max": 200},
        "H2": {"min": 200}
      }
    },
    {
      "sex": "female",
      "age_min": 18,
      "age_max": null,
      "pregnancy": false,
      "zones": {
        "L2": {"max": 70},
        "L1": {"min": 70, "max": 119},
        "L0": {"min": 120, "max": 160},
        "H1": {"min": 160, "max": 185},
        "H2": {"min": 185}
      }
    }
  ]
}
```

*Источники референсов:* [ICSH 2014] для общих диапазонов; [WHO 2011] для анемии (HGB <130/120 г/л); [BCSH 2017] для тромбоцитов; популяционные данные NHANES, GeKid, Российские методические рекомендации [НИИ гематологии, 2020].

### 3.2 `patterns.json` – синдромальные шаблоны

Каждый паттерн описывается:
- `id` – уникальный идентификатор (например, `PANCYTOPENIA`, `HEMOLYSIS_PATTERN`)
- `match` – логическое дерево (AND, OR, NOT), каждый лист: `{param, zones[]}`
- `severity` – «red» (must-not-miss), «amber», «green»
- `differentials` – список нозологий, передаваемых в DiffDiagnosis
- `refs` – клинические ссылки

Пример:

```json
{
  "id": "PANCYTOPENIA",
  "match": {
    "AND": [
      {"param": "WBC",   "zones": ["L1","L2"]},
      {"param": "HGB",   "zones": ["L1","L2"]},
      {"param": "PLT",   "zones": ["L1","L2"]}
    ]
  },
  "severity": "red",
  "differentials": [
    "aplastic_anemia", "MDS", "myelofibrosis", "B12_deficiency",
    "hypersplenism", "PNH", "bone_marrow_infiltraton"
  ],
  "refs": ["Hoffbrand, 2019, ch.9", "Williams Hematology, 9e, ch.30"]
}
```

Для парных паттернов (например, Bessman classification) используется комбинация двух параметров с проверкой по зонам:

```json
{
  "id": "ANEMIA_MICROCYTIC_SET",
  "match": {
    "AND": [
      {"param": "HGB", "zones": ["L1","L2"]},
      {"param": "MCV", "zones": ["L1","L2"]},
      {"param": "MCH", "zones": ["L1","L2"]},
      {"param": "RDW", "zones": ["H1","H2"]}
    ]
  },
  "severity": "amber",
  "differentials": ["iron_deficiency"],
  "alternative": {
    "AND": [
      {"param": "HGB", "zones": ["L1","L2"]},
      {"param": "MCV", "zones": ["L1","L2"]},
      {"param": "RBC", "zones": ["H1","H2"]},
      {"param": "RDW", "zones": ["L0"]}
    ],
    "differentials": ["thalassemia_trait"]
  }
}
```

## 4. Слой Engine (`ssa-engine`)

Реализован на Rust (crate). Основные структуры:

- `CbcInput` – содержит значения 28 параметров + демография (sex, age, pregnancy).
- `ZoneVector` – массив `Option<Zone>` длины 28. `Zone` – `enum { L2, L1, L0, H1, H2 }`.
- `MatchedPattern` – содержит `pattern_id`, `severity`, `score` (вероятность/вес).
- `RankedSyndromeList` – упорядоченный по убыванию `score` список.

### 4.1 Digitize

```rust
fn digitize(cbc: &CbcInput, refs: &RangesCollection) -> Result<ZoneVector, Error>
```

1. По полу, возрасту, статусу беременности выбирается подходящая запись для каждого из 28 параметров.
2. Число сравнивается с интервалами каждой зоны; попадание в один интервал -> соответствующая зона.
3. Если значение на границе (например, равно порогу L1/L0), используется правило «приоритет патологической зоны» (L2 > L1 > L0 < H1 < H2) — в клинической практике пограничные значения трактуются в сторону отклонения.

### 4.2 Matcher

```rust
fn match_patterns(vector: &ZoneVector, patterns: &[Pattern]) -> Vec<MatchedPattern>
```

Рекурсивный спуск по логическому дереву `match`. Поддержка:
- `AND` – истина, если все дети истинны.
- `OR` – истина, если хотя бы один ребёнок истинен.
- `NOT` – истина, если ребёнок ложен.
- Лисья проверка: `param` присутствует в зоне, если его зона входит в `zones[]`.

Для каждого сработавшего шаблона вычисляется `score`. Простейший вариант: для `AND`-паттерна количество совпадений зон, нормированное на общее число листьев. Более точный – байесовский вес на основе априорной распространённости синдрома (данные литературы или онтологии).

### 4.3 Ranker

```rust
fn rank(matched: Vec<MatchedPattern>) -> Vec<MatchedPattern>
```

- Первичная сортировка по `severity`: red > amber > green.
- Внутри группы – по убыванию `score`.
- Порог отсечки: удаляются паттерны с `score` ниже 0.3 (настраиваемо).

## 5. Слой API (`ssa-api`, axum)

| Endpoint | Method | Input | Output | Примечание |

## 6. Слой UI (Phoenix LiveView)

- **Страница `/cbc`**: форма с полями для 28 параметров (числовые вводы) + выбор демографии (пол, возраст, беременность). Кнопка «Анализировать».
- **Страница `/cbc/:id`**: результат – таблица параметров с цветной индикацией зоны (красный/оранжевый/зелёный) + ранжированный список синдромов с дифференциалами и ссылками на клиническую классификацию.
- **Страница `/parameters`**: список всех 28 параметров с базой референсов (возможность фильтрации по демографии).
- **Страница `/patterns`**: каталог всех 30 + 15 = 45 паттернов с описанием и референсами.

При реализации учитывать принципы *progressive enhancement*: при отключённом JS остаётся статический REST-интерфейс.

## 7. Тестирование

### 7.1 Gold-стандарт

Формируется набор из 20+ клинически верифицированных кейсов (CBC + подтверждённый диагноз). Примеры:

| № | CBC-значения | Ожидаемый синдром | Диагноз |
|---|--------------|-------------------|---------|
| 1 | WBC 1.2, HGB 65, PLT 18 | PANCYTOPENIA | Апластическая анемия |
| 2 | WBC 35, PLT 750, HGB 185 | MYELOPROLIFERATIVE | Истинная полицитемия |
| … | … | … | … |

### 7.2 Метрики

- **Top-1 accuracy** – доля кейсов, где истинный синдром находится на первой позиции ранжированного списка.
- **Top-3 accuracy** – доля, где истинный синдром в первых трёх.
- **Red-flag-miss-rate** – доля случаев с паттернами `severity: "red"`, которые не попали в топ-10 (должно быть 0).
- **Calibration (
