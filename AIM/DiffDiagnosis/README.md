# DiffDiagnosis

Дифференциально-диагностический движок, объединяющий классические алгоритмы русской и англо-американской терапевтических школ с современными методами байесовского вывода, машинного обучения и когнитивной психологии. Предназначен для помощи клиницисту в построении и сужении дифференциального ряда, оценке вероятностей гипотез и минимизации диагностических ошибок.

## Зачем

Диагностические ошибки остаются одной из главных проблем клинической медицины. Ограниченная широта дифференциального ряда, когнитивные искажения (якорение, преждевременное закрытие) и неполнота параклинической информации ведут к пропущенным диагнозам, отсрочке лечения и увеличению смертности. DiffDiagnosis предлагает структурированный, доказательный и проверяемый подход:

- детерминированные алгоритмы на основе канонических источников (Виноградов, Taylor);
- вероятностное обновление гипотез (байесовская модель, отношения правдоподобия);
- интеграцию клинических правил (decision rules) и EBM-рекомендаций;
- обнаружение когнитивных ловушек и принудительное расширение ряда (cognitive forcing).

## Как устроено

Высокоуровневая архитектура включает три слоя:

1. **Backend (Rust, axum + serde)** — детерминированный диагностический движок, JSON‑схемы алгоритмов из `sources/`, байесовский калькулятор, роутер к LLM‑уровню для интерпретации свободного текста и генерации отчётов.
2. **Frontend (Phoenix LiveView)** — интерактивный интерфейс для ввода жалоб, анамнеза и параклиники, отображения дифференциального ряда с вероятностями, пошаговых рекомендаций и визуализации диагностического пути.
3. **LLM‑уровень (DeepSeek через AIM‑роутер)** — обработка неструктурированного текста, сопоставление с illness scripts, генерация нарративных резюме; не участвует в принятии решений, а лишь дополняет детерминированный движок.

## Запуск backend (Rust)

```bash
cd backend
cargo run
```

Сервер запускается на `http://127.0.0.1:8080`. Убедитесь, что установлен Rust (stable) и Cargo.

## Запуск frontend (Phoenix)

```bash
cd frontend
mix deps.get
mix phx.server
```

Приложение будет доступно на `http://127.0.0.1:4000`. Требуется Erlang/OTP 26+, Elixir 1.15+.

## API

Основные REST endpoints (Swagger/OpenAPI спецификация доступна после запуска backend):

| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/api/diagnosis/chest-pain` | Запуск алгоритма «Боль в груди» (Vinogradov). |
| `POST` | `/api/diagnosis/dyspnea` | Алгоритм «Одышка / кашель / кровохарканье». |
| `POST` | `/api/diagnosis/abdominal-pain` | Синдром «Острый живот». |
| `POST` | `/api/diagnosis/jaundice` | Желтуха. |
| `POST` | `/api/diagnosis/fever` | Лихорадка / FUO. |
| `POST` | `/api/diagnosis/lymphadenopathy` | Лимфаденопатия / спленомегалия. |
| `POST` | `/api/diagnosis/anemia` | Анемия / цитопении. |
| `POST` | `/api/bayesian/update` | Байесовское обновление вероятностей по заданному тесту. |
| `POST` | `/api/llm/interpret` | Интерпретация свободного текста жалоб (DeepSeek). |

Формат запроса/ответа — JSON.

## Источники

- Виноградов А.В. «Дифференциальная диагностика внутренних болезней» (3-е изд., 2001) — основные алгоритмы для боли в груди, одышки, острого живота, желтухи, лихорадки, лимфаденопатии, анемий.
- Taylor R.B. «Difficult Diagnosis» (2nd ed., 1992) — диагностические дилеммы при нормальных коронарных артериях, резистентной гипертензии, хроническом кашле, узле лёгкого, плевральном выпоте, хронической абдоминальной боли, ЖКК неясного генеза, отклонениях печёночных ферментов, АКИ, гематурии, протеинурии, гипонатриемии, узле щитовидной железы, цитопениях, FUO, сепсисе без источника.
- Evidence-based guidelines (ESC, ACCP, ATS, BSG, KDIGO) — клинические правила (PERC, Wells, CURB-65, MELD, CHA₂DS₂‑VASc и др.), интегрированные в детерминированный движок.
- Мета-аналитический обзор (meta_*.md) — байесовский подход, модели Pauker–Kassirer, Dual-Process Theory, классификация диагностических ошибок (Graber, Newman-Toker), когнитивные forcing strategies.
